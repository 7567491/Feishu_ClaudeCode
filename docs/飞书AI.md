# 飞书AI项目汇总（GitHub Stars > 1000）

本文档汇总了GitHub上与飞书集成的热门AI项目，所有项目的星标数均超过1000。

## 1. chatgpt-on-wechat

**GitHub仓库**: zhayujie/chatgpt-on-wechat
**Stars**: 25,800+ ⭐
**主要语言**: Python

### 项目简介
基于大模型搭建的多平台聊天机器人，同时支持微信公众号、企业微信应用、**飞书**、钉钉等接入。

### 核心特性
- **多模型支持**: GPT-3.5/GPT-4o/GPT-4/Claude/文心一言/讯飞星火/通义千问/Gemini/GLM-4/Kimi/LinkAI
- **多模态处理**: 能处理文本、语音和图片
- **系统集成**: 访问操作系统和互联网
- **企业定制**: 支持基于自有知识库进行定制企业智能客服

### 应用场景
适合需要在多个平台部署统一AI客服的企业，支持知识库定制化。

## 2. LangBot

**GitHub仓库**: langbot-app/LangBot
**Stars**: 14,200+ ⭐
**主要语言**: Python
**官方文档**: https://docs.langbot.app

### 项目简介
面向LLM时代设计的全球IM机器人平台，支持QQ、Discord、LINE、WeChat、Telegram、**飞书**、DingTalk、Slack等多个平台。

### 核心特性
- **丰富的AI集成**: ChatGPT、DeepSeek、Dify、n8n、Langflow、Coze、Claude、Google Gemini、xAI、PPIO、Ollama、MiniMax、SiliconFlow、Qwen、Moonshot
- **MCP支持**: 支持Model Context Protocol（模型上下文协议）
- **LLM、Agent、RAG**: 支持检索增强生成等高级功能
- **开源协议**: Apache-2.0

### 应用场景
适合需要在多个国际化平台部署AI机器人的团队，支持主流的LLM服务。

## 3. Feishu-OpenAI

**GitHub仓库**: ConnectAI-E/feishu-openai
**Stars**: 5,600+ ⭐
**主要语言**: Go

### 项目简介
飞书 × OpenAI的深度集成项目，提供"飞一般的工作体验"。集成了GPT-4、GPT-4V、DALL·E-3、Whisper等OpenAI核心服务。

### 核心特性
- **🗣 语音交流**: 基于Whisper的语音识别与对话
- **👁️ 图像理解**: GPT-4V支持图片分析
- **🎨 图片创作**: DALL·E-3文生图功能
- **🎭 角色扮演**: 预配置的AI角色和人设模式
- **💬 多话题讨论**: 支持私聊和群聊的上下文保持
- **📊 表格分析**: 数据分析能力
- **📄 文档导出**: 富文本卡片响应和文档生成
- **💰 使用统计**: Token使用跟踪和余额查询

### 部署方式
- Serverless部署
- Docker容器部署
- 本地部署

### 应用场景
专为飞书深度定制的OpenAI集成方案，适合重度使用OpenAI服务的团队。

## 飞书文档与多维表格专项AI项目

以下项目专注于飞书文档（Docs）和多维表格（Bitable）的AI增强功能。

### 6. FastGPT ⭐ 推荐

**GitHub仓库**: labring/FastGPT
**Stars**: 26,400+ ⭐ (所有项目中最高)
**主要语言**: TypeScript (NextJS)
**开源协议**: Apache-2.0

#### 项目简介
基于LLM的知识库问答系统和AI Agent构建平台，提供开箱即用的数据处理、RAG检索和可视化AI工作流编排能力。**原生支持飞书集成**。

#### 核心特性
- **📚 知识库管理**: 支持多种文件格式导入（txt、pdf、docx、md、csv等）
- **🔍 RAG检索**: 混合检索、重排序，提供高质量问答
- **🎨 可视化工作流**: 通过Flow界面进行应用设计和编排
- **🔌 OpenAPI接口**: 对齐OpenAI官方API格式，易于集成
- **🤖 多模型支持**: 可配置不同的LLM模型
- **🚀 飞书深度集成**:
  - 支持飞书机器人接入
  - 可将飞书文档导入知识库
  - 提供完整的接入教程和视频

#### 与飞书文档/多维表格结合方式
- 将飞书文档内容导入FastGPT知识库
- 基于飞书文档内容构建AI问答机器人
- 在飞书中直接调用AI知识库能力
- 支持将分析结果回传到飞书

#### 应用场景
- 企业知识库智能问答
- 飞书文档智能检索
- 客服机器人
- 文档内容自动总结和分析

### 7. feishu2md

**GitHub仓库**: Wsine/feishu2md
**Stars**: 1,800+ ⭐
**主要语言**: Go
**开源协议**: MIT

#### 项目简介
一键下载飞书文档为Markdown的命令行工具，支持单文档、文件夹和知识库的批量导出。

#### 核心特性
- **📥 单文档转换**: 将飞书文档链接转换为Markdown格式
- **📦 批量下载**: 支持下载整个文件夹内的全部文档
- **📚 知识库导出**: 可批量下载知识库中的所有文档
- **🖥️ 多种使用方式**:
  - 命令行工具（Windows、macOS、Linux）
  - Docker容器版本
  - 在线Web版本
- **🔧 灵活配置**: 支持飞书API Token配置
- **📎 附件下载**: 支持下载文档中的附件

#### 与AI结合的应用场景
虽然本身不是AI项目，但作为飞书文档的重要工具，可以：
- **为AI训练准备数据**: 将飞书文档批量导出为Markdown，作为训练语料
- **接入RAG系统**: 导出的Markdown可直接导入FastGPT等知识库系统
- **文档备份**: 定期备份飞书知识库内容
- **跨平台迁移**: 将飞书文档迁移到其他支持Markdown的平台

#### 典型工作流
```
飞书文档/知识库 → feishu2md导出 → Markdown文件 → FastGPT导入 → AI知识库
```

### lark-openapi-mcp (官方项目)

**GitHub仓库**: larksuite/lark-openapi-mcp
**Stars**: 293 ⭐
**开发者**: 飞书官方团队

#### 项目简介
飞书官方推出的OpenAPI MCP工具，将飞书开放平台API封装为Model Context Protocol工具，允许AI助手（如Claude、Cursor）直接调用飞书接口。

#### 核心特性
- **官方支持**: 飞书团队维护，接口更新及时
- **文档处理**: 支持文档读写、编辑
- **多维表格操作**: 支持Bitable的增删改查
- **对话管理**: 管理飞书会话
- **日历安排**: 自动化日历操作
- **双域名支持**: 国内版和国际版Lark
- **灵活认证**: 应用级和用户级OAuth认证

#### 应用场景
- 让Claude等AI直接操作飞书文档和多维表格
- AI自动化工作流（如自动生成周报、数据分析）
- 在IDE中通过AI助手操作飞书

## 更新后的选型建议

### 按使用场景分类

#### 📄 文档和知识库场景
1. **构建企业知识库**: **FastGPT** (支持飞书文档导入、RAG检索)
2. **文档批量导出**: **feishu2md** (Markdown格式导出)
3. **文档+知识库组合方案**: feishu2md导出 → FastGPT导入

#### 📊 多维表格场景
1. **表格数据智能分析**: **Feishu-OpenAI-Base-Helper** (数据向量化和问答)
2. **AI直接操作表格**: **lark-openapi-mcp** (官方MCP支持)
3. **表格数据自动化**: 配合FastGPT构建数据分析工作流

#### 💬 聊天机器人场景
1. **飞书专用**: **Feishu-OpenAI** (OpenAI全家桶)
2. **多平台部署**: **chatgpt-on-wechat** 或 **LangBot**
3. **创意生成**: **MuseBot** (图片/视频)

#### 🛠️ 开发者场景
1. **深度定制**: **Koishi** (TypeScript框架)
2. **AI工具集成**: **lark-openapi-mcp** (在Cursor/Claude中操作飞书)
3. **知识库开发**: **FastGPT** (可视化工作流)

### 推荐组合方案

#### 方案一：企业知识管理全栈
```
飞书文档 → feishu2md → FastGPT知识库 → 飞书机器人
```
适合：希望将飞书文档转化为可搜索、可问答的智能知识库

#### 方案二：多维表格智能分析
```
飞书多维表格 → Feishu-OpenAI-Base-Helper → AI分析报告 → 飞书通知
```
适合：需要对业务数据进行定期分析和报告

#### 方案三：AI工作流自动化
```
lark-openapi-mcp + FastGPT → AI自动操作飞书文档和表格
```
适合：开发者希望通过AI助手自动化日常工作
